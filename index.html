Circles Learning Labs

Fullstack Engineer and Dev-ops • Full-time, Remote (February 2022 - November 2023)
NextJs • TypeScript • React • Erlang • Elixir • Jest • Testing Library React • GraphQL • Postgres • DynamoDB • AWS

When I started, the existing frontend product was an amalgimation of vanilla JavaScript, JQuery, and React bundled into the backend project. The project was a WebRTC based video conferencing platform with many user interactions and features. The total page load was something like 18mb and the build/deploy time took 30 mins on a good day.
The first major project I was a part of was rewriting the frontend using modern web standards. We chose NextJs and TypeScript for the frontend for the speed and type safety it provided. The backend was written in Elixir and Erlang and we used GraphQL to communicate between the two. This was mostly complete already so my focus was on the frontend. We cut the total page load down to 2mb and added in a host of basic accessibility and localization options.
Circles had the barebones of a component library for use in this weird amalgimation of frontend technologies but once the rewrite project began we could start to finalise the library using good practices, modern standards, and a focus on accessibility. We also started using Storybook to document the components and make it easier for the team to use them. This rebuild also included a big push for unit tests on components using Jest and Testing Library React.
As we gained more traction in our optimization efforts, we started to look at the CI/CD pipeline and how we could improve it. We moved from a single monolithic pipeline to a more modular approach and did a major overhaul on the Dockerfiles and Caching strategies to speed up the build times. We cut the build and deploy times in half, which will save many hours of developer time in the long run.
The second major project I worked on while at Circles was the implementation of meeting Transcription and Live Captioning using AWS Chime. Over three months, I worked in a small team with one other coworker to interface with the clients directly and bring their feature visions to life.

GovHack New Zealand • 2020

Participant • Getting Around Challenge Runner Up

During the weekend hackathon, our team developed and demoed a tool that would help communities use open source and community data to help with improvements we called LivCity.
LivCity is an application that allows a city to gain community sourced information on selected areas.
This app functions by an administrator or researcher selecting an area to question, this could be for example regarding air pollution, traffic, safety, waste and more.
Using feedback from the community, the appropriate authorities can develop needed infrastructure in areas or appropriate social services, greatly improving the efficiency, usefulness, and bringing innovation to the community.

GovHack New Zealand • 2019

Participant • Innovate New Zealand Award • Best use of data in a Civil Emergency Award

For GovHack 2019 the team created a tourism app designed to help connect visitors to local businesses and help guide tourists to safe areas in the event of a civil emergency.
Localé is a tourism based application designed to better connect visitors to local businesses and events in a safe and stress free manner.
Localé will also include local area safety and evacuation information to improve community safety.
Data generated can be fed back to the council to aid in council and civil defense planning.
The end goal of our idea is to have a well designed app with a great user experience, while accomplishing our core concepts well.
